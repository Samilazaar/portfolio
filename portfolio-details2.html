Virgule

Virgule est une plateforme de mise en relation de personnes dans le cadre d’envoi de lettres au format « papier ». La plateforme permet de « matcher » des profils en fonction de centres d’intérêt renseignés.

Fonctionnement

Connexion

Nouvel utilisateur

L’utilisateur créé tout d’abord son compte :

  • Email ou numéro de téléphone
  • Mot de passe

    • Le mail est ensuite vérifié via un lien de validation envoyé sur la boite mail de l’utilisateur ou un code envoyé sur le numéro de téléphone.

    Le mot de passe pourra ensuite être modifié

    Une fois le compte validé, l’utilisateur peut renseigner son profil :

  • Nom
  • Prenom
  • Adresse postale

    • Dans un premier temps l’adresse postale est composée de champs à renseigner librement

  • Numéro de voie
  • Type de voie (rue, avenue, boulevard, etc….)
  • Ville
  • Code postal
  • Pays
  • Centre d’intérêts

    • Une liste de choix multiples (ex : politique, sport, philosophie, poésie, cinéma, …)

    Ces éléments pourront ensuite être modifiés dans le profil de l’utilisateur

    Utilisateur existant

    Renseigner le mail ou téléphone et le mot de passe avec la possibilité de cliquer sur « mot de passe oublié » pour changer son mot de passe via un mail envoyé sur l’adresse mail de l’utilisateur ou par sms.

    Modification des informations du compte

    L’utilisateur, une fois connecté, peux à tout moment modifier ses infos

  • Mot de passe
  • Adresse postale
  • Centres d’intérêts

    • Recherche de correspondant

    L’utilisateur peux, une fois par jour (ou semaine, à définir) rechercher une personne dans la plateforme. Celui-ci clique sur « Nouvelle Virgule » et l’écran affiche une animation d’adresse (fictives) qui défile rapidement avant de lui afficher l’adresse d’un autre utilisateur ayant au moins un centre d’intérêt commun. L’interface affiche les informations de la personne et enregistre en base de données le fait que cet utilisateur a eu accès aux informations de ce profil avec l’heure d’accès. Si l’utilisateur a déjà fait une recherche ce jour (ou cette semaine), l’interface affiche un message lui indiquant qu’il a déjà fait une recherche en rappelant les infos de la dernière recherche et lui demandant de patienter (en affichant le temps restant jusqu’à la recherche suivante).

    Mes correspondants

    Cette interface affiche la liste des correspondants affichés sur les recherches de l’utilisateur en triant de la plus récente à la plus ancienne. Elle affiche également la liste des personnes ayant eu accès à mon profil via la recherche (toujours du plus récent au plus ancien).

    Dans cette liste l’utilisateur peut :

  • Indiquer qu’il n’a jamais reçu de courrier de cette personne
  • Signaler un courrier « indésirable » (contenu inapproprié)
  • Bloquer un utilisateur ( Celui-ci n’a donc plus accès au profil et une mention « utilisateur inconnu» apparait dans la liste de ses correspondants )
  • Archiver un profil
  • Marquer en favori une des personnes
  • Décerner un « badge » spécial à une personne

    • Partie technique

    La plateforme sera, dans un premier temps, sous la forme d’une plateforme web. Pour le développement du code soit en PHP soit ASP.net (en MVC dans les 2 cas). Pour la base de donnée, le format est libre (MongoDB ou base de donnée relationnelle). Pour la sauvegarde et la mise en production, utiliser Bitbucket via Git Une branche master (ou production) qui sera celle utilisée en ligne Une branche staging (ou développement) qui sera celle utilisée pour les tests avant passage en production) Une branche ensuite par partie de développement (chaque branche correspond à un ticket jira) Chaque publication dans la branche staging ou master doit se faire via une pull request dans bitbucket en ajoutant « Abd El Majid KALLED » comme examinateur. Chaque document doit également être ajouté au wiki.
